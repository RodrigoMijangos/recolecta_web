# Plantilla Universal de Issues

Usa esta plantilla para reportar bugs, solicitar features o crear tareas generales. Copia y pega el contenido según necesites.

---

assignees: ""
---

## Secciones principales

| Sección | Propósito |
|---------|-----------|
| **Resumen** | Descripción clara y breve del problema o solicitud |
| **Tipo de issue** | Clasifica el tipo: Bug, Feature, Docs, Task o Pregunta |
| **Pasos para reproducir** | Pasos específicos para reproducir el problema (solo si es bug) |
| **Comportamiento esperado** | Qué debería ocurrir |
| **Comportamiento observado** | Qué ocurre realmente |
| **Entorno** | Sistema operativo, navegador, versión del proyecto, dependencias |
| **Logs y evidencias** | Logs, errores, capturas de pantalla, GIFs |
| **Reproducir localmente** | Comandos o pasos para reproducir en local |
| **Criterios de aceptación** | Requisitos a cumplir (para features y tareas) |
| **Impacto y prioridad** | Evaluación del impacto y prioridad sugerida |
| **Issues/PRs relacionados** | Enlaces a otros issues o PRs relevantes |
| **Checklist** | Verificación antes de crear el issue |

## Notas de uso

- Completa solo las secciones relevantes para tu tipo de issue
- Usa etiquetas claras: `bug`, `feature`, `enhancement`, `documentation`, `question`, etc.
- Asigna prioridad: `priority-high`, `priority-medium`, `priority-low`
- Vincula issues relacionados usando `#numero-del-issue`
